# Android v0.2.0 真机验收清单

这份清单用于验证本轮 Android 版本的核心交付是否达到可发布水平。

覆盖范围：

- 主流程双语
- 锁屏/后台恢复
- 语音输入与录音胶囊
- 相册 / 拍照附件
- Git 操作扩展
- 后台完成通知

建议测试设备：

- Android 13 或更高 1 台
- Android 11 或 12 1 台
- 最少 1 台中文系统
- 最少 1 台英文系统

建议测试环境：

- 同一局域网内稳定 Wi‑Fi
- 可切换蜂窝或关闭网络模拟弱网
- 目标仓库存在 Git remote，方便验证 `push / pull`

## 一、安装与基础启动

1. 安装最新 APK。
   通过标准：应用可正常安装，桌面图标和应用名显示正常。

2. 冷启动应用。
   通过标准：启动页无崩溃；中文系统显示中文文案，英文系统显示英文文案；不存在中英混写。

3. 进入服务器列表、登录、会话列表。
   通过标准：主流程页面文案与系统语言一致；无明显硬编码中文残留。

## 二、双语与基础文案

1. 在中文系统下完整走一遍：
   服务器列表 -> 登录 -> 会话列表 -> 会话详情 -> 收件箱 -> 设置。
   通过标准：核心页面文案均为中文。

2. 在英文系统下完整走一遍同样路径。
   通过标准：核心页面文案均为英文；按钮、空状态、错误提示、弹窗标题无中文漏出。

3. 检查会话详情中的状态文案：
   运行中、已完成、失败、已停止、同步中、恢复中。
   通过标准：状态文案与系统语言一致，时间与日期格式合理。

## 三、锁屏 / 后台恢复

1. 进入会话详情，发送一条会触发较长运行的消息。
   在运行过程中按电源键锁屏 1 分钟，再解锁。
   通过标准：页面能自动恢复；不出现误导性的红色报错；输出可继续补齐。

2. 同样场景下锁屏 5 分钟。
   通过标准：回前台后自动刷新当前 run 与会话内容；若运行结束，页面落到最终状态。

3. 锁屏 10 到 30 分钟。
   通过标准：即使实时流断过，也应表现为“恢复中/已同步”而不是强失败；恢复提示是中性提示，不是错误心智。

4. 切后台后切换网络：
   Wi‑Fi -> 蜂窝，或断网再恢复。
   通过标准：回前台后可自动重连或补齐；如果确实不可恢复，才显示错误。

## 四、后台完成通知

1. Android 13+ 首次进入会话详情。
   通过标准：会请求通知权限；拒绝后不崩溃；允许后后续可发通知。

2. 进入会话详情，发起运行，然后切到后台等待任务完成。
   通过标准：收到“运行已结束”本地通知。

3. 点击通知。
   通过标准：应用被唤起并尽量回到对应会话详情页。

4. 同一 run 结束后重复进入前后台。
   通过标准：不会对同一 run 重复刷多条通知。

5. 分别验证：
   `completed`、`failed`、`stopped`
   通过标准：通知文案能区分不同结束状态。

## 五、会话详情加载与恢复动效

1. 首次打开一个会话详情页。
   通过标准：能看到分阶段加载态：
   - 连接主机
   - 恢复实时流
   - 整理会话内容

2. 从后台恢复进入会话详情。
   通过标准：出现轻量恢复提示；动效自然，不突兀，不挡住核心内容。

3. 检查滚动与输入。
   通过标准：加载态和恢复提示不影响滚动、输入和底部 composer 使用。

## 六、语音输入与录音胶囊

1. 首次点击麦克风。
   通过标准：请求麦克风权限；拒绝后给出可理解提示；允许后进入录音态。

2. 录音中观察胶囊。
   通过标准：看到录音胶囊、波形条、时长、停止/取消入口。

3. 点击“停止”。
   通过标准：进入转写中状态，随后把文本写回输入框。

4. 点击“取消”。
   通过标准：录音态消失，不会把脏数据写回输入框。

5. 说一段中文。
   通过标准：中文识别结果能正常进入输入框。

6. 说一段英文。
   通过标准：英文识别结果能正常进入输入框。

7. 说话很短、很轻，或保持沉默。
   通过标准：出现“未识别到清晰内容”之类的兜底提示，不崩溃。

8. 录音中切后台或锁屏。
   通过标准：录音会被安全取消或安全收尾，不应把 UI 留在坏状态。

## 七、附件入口：文件 / 相册 / 拍照

1. 文件选择器上传。
   通过标准：原有文件上传仍可用，未回归。

2. 相册选择图片。
   通过标准：可成功选图；草稿会话进入本地附件队列；已有会话直接上传或进入附件列表。

3. 拍照上传。
   通过标准：相机可拉起；拍照后返回应用；图片进入附件链路。

4. 拍照后取消。
   通过标准：不会残留异常附件，不崩溃。

5. 在草稿会话中测试：
   文件 / 相册 / 拍照
   通过标准：都进入本地附件队列；发送首条消息后附件能被带上。

6. 在已有会话中测试：
   文件 / 相册 / 拍照
   通过标准：都能进当前会话附件链路；可正常发送。

## 八、Git 操作扩展

建议在一个干净且已配置 remote 的测试仓库中完成。

1. 查看 Repo 状态卡片。
   通过标准：能看到分支、根目录、ahead/behind、staged/unstaged/untracked 等信息。

2. `createBranch`
   通过标准：成功创建并切换。

3. `checkout`
   通过标准：成功切换目标分支。

4. `commit`
   通过标准：有改动时可提交；无改动时给出合理提示。

5. `push`
   通过标准：当前分支可推送到远端。

6. `pull`
   通过标准：能拉取远端最新提交；如果不能快进，提示合理。

7. `stash`
   通过标准：可暂存当前改动，包含未跟踪文件。

8. `log`
   通过标准：能看到最近提交记录；标题、作者、时间展示正常。

9. 在非 Git 目录打开会话。
   通过标准：不会暴露错误的 Git 控件；或操作时有明确提示。

## 九、异常与边界

1. 通知权限拒绝后继续使用应用。
   通过标准：只是不发通知，不影响主流程。

2. 麦克风权限拒绝后继续发送文字消息。
   通过标准：文字主流程不受影响。

3. 相机不可用或系统无图库。
   通过标准：出现可理解提示，不崩溃。

4. 仓库为 detached HEAD。
   通过标准：`push / pull` 等受限制操作提示明确。

5. 长会话详情页反复打开、滚动、切后台再回来。
   通过标准：没有明显卡顿、重复提示或状态错乱。

## 十、发布前结论

全部通过：

- 可进入发布准备。

存在少量非阻塞问题：

- 记录为已知问题，但需确认不影响主流程和数据安全。

存在以下任一问题：

- 会话恢复失败率高
- 通知点击不能稳定回流
- 语音输入经常不可用
- 拍照/相册链路不稳定
- Git 扩展存在误操作或错误结果

则建议：

- 暂不发布，先修复后复测。

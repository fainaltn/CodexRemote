# Android v0.4.0 真机验收清单

这份清单用于最后一轮统一实机测试，覆盖 `v0.4.0` 目前已经落地的 Android 主流程。

覆盖范围：

- 配对与可信重连
- 会话搜索与归档恢复
- 轻量 smart composer
- 设置页偏好与运行默认值
- 前台完成反馈与通知回流

建议测试设备：

- Android 13 或更高 1 台
- Android 11 或 12 1 台
- 至少 1 台中文系统
- 至少 1 台英文系统

建议测试环境：

- 同一局域网稳定 Wi‑Fi
- 可切换蜂窝或短暂断网模拟恢复
- 目标主机已跑起 `CodexRemote` 服务
- 至少一个测试仓库可正常创建 / 继续会话

## 一、安装与冷启动

1. 安装最新 APK。
通过标准：应用可正常安装，图标、应用名和版本信息显示正常。

2. 冷启动应用。
通过标准：启动页无崩溃；如果本机已有可信主机，能进入自动重连判断；如果没有，能看到明确的“去服务器列表 / 输入配对码”入口。

3. 中文系统与英文系统分别冷启动。
通过标准：启动页文案和状态文案随系统语言切换，无明显中英混写。

## 二、配对与可信重连

1. 在一台未配对手机上打开应用。
通过标准：可从启动页、服务器列表、添加服务器页、设置页进入配对入口。

2. 在主机侧生成一次性配对码。
通过标准：`npm run codexremote -- pair` 能返回配对码。

3. 在手机端输入配对码完成绑定。
通过标准：配对成功后自动保存主机、进入会话列表；下次启动不必重复输入。

4. 关闭应用后重新打开。
通过标准：如果可信凭据仍有效，应用优先尝试自动重连。

5. 在设置页关闭“自动重连”。
通过标准：可信记录仍保留，但下次启动不再直接自动重连。

6. 在设置页清除信任记录。
通过标准：下次启动回到未配对状态，且不会继续走可信重连。

## 三、会话列表、搜索与归档

1. 进入会话列表。
通过标准：项目分组、当前项目、排序与隐藏逻辑仍然正常。

2. 使用搜索框输入标题关键词。
通过标准：会话列表按标题筛选，项目分组结构仍然保留。

3. 使用搜索框输入路径关键词。
通过标准：能按 `cwd` / 项目路径筛选。

4. 归档一个会话。
通过标准：主列表中该会话消失，并出现成功反馈。

5. 打开“归档会话”页。
通过标准：能看到刚归档的会话，支持搜索。

6. 恢复归档一个会话。
通过标准：恢复后该会话从归档页消失，并重新回到主列表。

## 四、Smart Composer

1. 在会话详情中输入 `/`。
通过标准：会出现紧凑命令候选条，不遮挡主窗。

2. 选中 `/status`。
通过标准：直接触发原生状态/诊断路径，而不是只把文字塞进输入框。

3. 选中 `/archive`、`/pair`、`/refresh`、`/stop`、`/new`。
通过标准：这些高频命令都走原生动作；在不适用状态下不应引发崩溃。

4. 输入 `@`。
通过标准：先看到当前工作区首屏文件 / 目录候选。

5. 点一个目录，再点 `..` 返回。
通过标准：目录下钻与返回都正常，候选条保持轻量。

6. 输入 `@` 后继续输入关键词。
通过标准：切到文件搜索结果，结果与工作区实际文件一致。

7. 输入 `$`。
通过标准：能看到技能候选。

8. 切换 `$` 的来源筛选。
通过标准：全部 / 仓库技能 / 用户技能 三种来源切换正确。

9. 选中任意 `@` 或 `$` 候选。
通过标准：只是在现有输入框里插入 token，不会打开新的编辑页，也不会让 composer 膨胀。

## 五、设置页

1. 打开设置页。
通过标准：可看到外观、通知、运行默认值、配对与重连、密码修改等 section。

2. 点击“切换主题”。
通过标准：主题在 `AUTO / LIGHT / DARK` 间切换，图标和文案随状态变化。

3. 点击“管理系统通知”，切到系统设置再返回。
通过标准：设置页能刷新通知状态。

4. 修改“运行默认值”中的模型与思考强度。
通过标准：保存后会有成功反馈。

5. 点击“恢复自动默认值”。
通过标准：模型与思考强度回到自动。

6. 检查配对与重连 section。
通过标准：若已配对，能看到信任标签、配对方式、首次配对时间、最近自动重连时间。

## 六、运行默认值生效

1. 在设置页把默认模型 / 思考强度改成非自动。
通过标准：返回后，新开的草稿线程会优先带入这些默认值。

2. 新建或继续一个没有显式改过 runtime 的会话。
通过标准：默认值能自动带入。

3. 手动修改当前会话的 runtime。
通过标准：当前会话内的显式选择优先，不被设置页默认值覆盖。

## 七、前台反馈与通知

1. 在前台发起一轮会话运行并等待完成。
通过标准：完成后出现轻量会话内反馈卡片或反馈提示。

2. 让运行失败。
通过标准：出现失败反馈，不会只有静默状态变化。

3. 手动停止当前运行。
通过标准：出现“已停止”反馈。

4. 在前台执行 repo 操作。
通过标准：成功摘要会通过统一反馈层呈现。

5. 切到后台等待完成通知。
通过标准：如果权限开启，仍能收到系统通知并可回流到对应会话。

## 八、回归检查

1. 语音输入。
通过标准：权限、录音、转写与回填输入框仍然正常。
当前备注：三星 Z Fold7 / Android 16 路径仍需在下一个小版本继续修正，`v0.4.0` 先作为已知跟进项保留。

2. 文件 / 相册 / 拍照附件。
通过标准：已有附件链路无回归。

3. 会话详情滚动、恢复、弱网刷新。
通过标准：恢复态、降级态、前台自动补齐仍可用。

4. 仓库操作。
通过标准：分支、提交、推送、拉取、stash、日志入口无明显回归。

## 九、发布前结论

全部通过：

- 可以进入发布准备。

存在少量非阻塞问题：

- 记录为已知问题，但需确认不影响主流程和数据安全。

以下任一问题仍存在，则建议暂缓发布：

- 可信重连经常失败
- 归档恢复不稳定
- Smart composer 候选经常插错 / 崩溃
- 设置页默认值保存后不生效
- 前台反馈与系统通知相互打架或重复过多
